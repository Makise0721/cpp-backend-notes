# 05 — 负载均衡与服务发现

> 关联篇目：[01 一致性哈希](./01-cap-consistent-hashing.md)

---

## 1. 负载均衡——把流量分到多台机器

### 1.1 L4（传输层）vs L7（应用层）

```
L4 负载均衡（TCP/UDP 级别）:
  Client → [L4 LB] → Server1, Server2, Server3
  LB 只看 IP:Port，不改包内容
  性能高，但无法根据 URL/Header 路由

L7 负载均衡（HTTP 级别）:
  Client → [L7 LB] → /api → Server Group A
                    → /images → Server Group B
  LB 解析 HTTP，可根据 URL/Host/Cookie 路由
  功能强，但性能略低
```

| | L4 | L7 |
|------|------|------|
| 工作层 | TCP/UDP | HTTP/HTTPS |
| 路由依据 | IP:Port | URL/Host/Header/Cookie |
| 性能 | 极高 | 高 |
| 适用 | 通用 TCP 服务 | HTTP 服务/API 网关 |

### 1.2 负载均衡算法

| 算法 | 原理 | 适用 |
|------|------|------|
| **轮询（Round Robin）** | 依次轮流分配 | 服务器性能相同 |
| **加权轮询** | 按权重分配（高性能机器权重高） | 异构服务器 |
| **最少连接** | 分配给当前连接数最少的 | 长连接（数据库连接池） |
| **IP Hash** | hash(client_ip) % N → 同 IP 始终打到同服务器 | 需要会话保持 |
| **一致性哈希** | 减少服务器增减时的重分配 | 分布式缓存 |

### 1.3 常见方案

```
硬件: F5（贵、性能强）
软件: Nginx、HAProxy、Envoy、LVS
云:   AWS ALB/NLB、GCP Cloud LB
```

## 2. 健康检查——自动摘除故障节点

```
主动检查: LB 定期请求 /health 端点
         超时/返回非 200 → 标记为不健康 → 不再分配流量

被动检查: 实际请求失败 → 累计到阈值 → 摘除
```

## 3. 服务发现——微服务怎么找到彼此？

### 3.1 问题

```
服务 A 要调用服务 B:
  硬编码 IP → B 扩容/迁移 → A 需要重启 → 不可接受

服务发现: A 问 "B 的地址是什么？" → 注册中心回答
```

### 3.2 客户端发现 vs 服务端发现

```
客户端发现:
  Service A → 注册中心 (查询 B 地址) → B1, B2, B3
  Service A → 自己负载均衡 → 选一个 B 调用

服务端发现:
  Service A → LB (固定地址) → 注册中心 → B1, B2, B3
  Service A 不需要知道 B 的具体地址
```

### 3.3 注册中心对比

| | Consul | Etcd | Nacos | Zookeeper |
|------|:---:|:---:|:---:|:---:|
| CAP | CP/AP 可选 | CP | CP+AP | CP |
| 健康检查 | ✅ | ❌ (需额外) | ✅ | ❌ (需额外) |
| KV 存储 | ✅ | ✅ (核心) | ✅ | ✅ |
| 多数据中心 | ✅ | ❌ | ✅ | ❌ |
| 生态 | HashiCorp | **K8s 核心** | Alibaba | Hadoop/大数据 |

## 4. 负载均衡 + 服务发现的标准架构

```
                    ┌─────────────┐
                    │ 注册中心      │ ← Consul/Etcd/Nacos
                    │ Service B   │
                    │ B1:10.0.0.1 │
                    │ B2:10.0.0.2 │
                    └──────┬──────┘
          注册              │ 查询
    ┌───────┴────────┐      │
    ▼                ▼      │
Service B1      Service B2  │
                            │
Service A ─────────────查询──┘ → 得到 B1, B2 → 客户端负载均衡
```

## 小结

| 概念 | 一句话 |
|------|--------|
| L4 vs L7 | IP:Port 级别 vs HTTP URL/Header 级别 |
| 一致哈希 | 扩容数据迁移最少，常用于缓存 LB |
| 健康检查 | 主动 /health + 被动失败计数 |
| 注册中心 | 服务注册 + 发现 + 健康检查 |
| 客户端发现 | 服务 A 自己从注册中心获取 B 的地址列表 |
