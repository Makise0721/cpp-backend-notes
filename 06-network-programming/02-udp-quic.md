## 1. UDP——无连接的轻量传输

### 1.1 特点

| 特性 | 说明 |
|------|------|
| 无连接 | 不握手，直接发 |
| 不可靠 | 不保证送达、不保证顺序、不保证不重复 |
| 无拥塞控制 | 不会因为网络拥塞而降低发送速率 |
| 低开销 | 头部仅 8 字节（TCP 头部 20-60 字节） |
| 支持广播/组播 | ✅（TCP 不行） |

### 1.2 UDP 报文格式

```
UDP 头部（8 字节）:
 0      16      32
┌───────┬───────┬───────────────┐
│ 源端口 │ 目的端口│  长度  │ 校验和 │
│ 2字节  │ 2字节  │ 2字节 │ 2字节 │
└───────┴───────┴───────┴───────┘
```

### 1.3 适用场景

| 场景 | 为什么用 UDP |
|------|-------------|
| **DNS** | 单次请求-响应，包小，TCP 握手开销太大 |
| **视频直播/语音** | 允许少量丢帧，但不能忍受 TCP 重传延迟 |
| **在线游戏** | 实时性 > 可靠性（位置信息丢了无所谓，下个包就更新了） |
| **IoT 传感器** | 数据量小、设备算力弱、偶尔丢几个采样可接受 |

## 2. QUIC——基于 UDP 的下一代传输协议

### 2.1 QUIC 解决的问题

**TCP 的痛点**：
- 三次握手 → 1-RTT 延迟
- TLS 握手 → 再加 1-2 RTT（HTTPS 最少 2-3 RTT 才能发数据）
- 队头阻塞（Head-of-Line Blocking）→ 丢一个包，后面的全被阻塞
- 连接迁移困难（换 IP 就得重连）

**QUIC 在 UDP 之上自己实现了可靠传输**：

```
         HTTP/3
    ┌──────────────┐
    │    QUIC       │ ← 可靠传输 + TLS 1.3 + 多路复用 + 连接迁移
    ├──────────────┤
    │     UDP       │ ← 无连接，负责发包
    ├──────────────┤
    │      IP       │
    └──────────────┘
```

### 2.2 0-RTT 握手

```
首次连接（1-RTT）:
Client → Server: ClientHello (含密钥交换)
Server → Client: ServerHello + 加密数据
         ↑ 合并了 TCP 握手 + TLS 握手

再次连接（0-RTT）:
Client → Server: 加密数据（用之前缓存的密钥）
         ↑ 零 RTT 就能发数据！
```

### 2.3 多路复用无队头阻塞

```
TCP (HTTP/2):  多个流复用一条 TCP 连接
  Stream 1: [A][B][C]
  Stream 2: [D][E][F]    ← TCP 丢包 → Stream 1 和 Stream 2 都被阻塞！

QUIC (HTTP/3): 每个流独立
  Stream 1: [A][B][C]    ← Stream 1 丢包 → 只阻塞 Stream 1
  Stream 2: [D][E][F]    ← Stream 2 不受影响
```

### 2.4 连接迁移

TCP 连接由四元组（src_ip, src_port, dst_ip, dst_port）标识 → 换网络（WiFi→4G）→ 必须重连。

QUIC 用 **Connection ID** 标识连接 → 换 IP 不影响 → 无缝迁移。

### 2.5 QUIC vs TCP

| | TCP + TLS 1.3 | QUIC |
|------|:---:|:---:|
| 首次握手 | 2 RTT | 1 RTT |
| 再次连接 | 1-2 RTT | **0 RTT** |
| 队头阻塞 | 有 | 无 |
| 连接迁移 | 不支持 | 支持 |
| 内核实现 | ✅ | ❌ 用户态（更快迭代） |
| 部署难度 | 低 | 中（UDP 可能被 QoS 限制） |

## 小结

| 概念 | 一句话 |
|------|--------|
| UDP | 无连接、不可靠、低开销、支持广播/组播 |
| QUIC | 基于 UDP 的可靠传输，0-RTT、多路复用无阻塞、连接迁移 |
| 为什么 UDP | DNS、视频、游戏——实时性 > 可靠性 |
| 为什么 QUIC | TCP + TLS 太慢 → 用 UDP 实现更快的可靠传输 |
