## 1. 网络分层模型

### 1.1 OSI 七层 vs TCP/IP 四层

```
OSI 七层              TCP/IP 四层         设备          数据单元
┌──────────┐        ┌──────────┐
│  应用层   │        │          │                         数据
│  表示层   │───────→│  应用层   │         —               (HTTP/DNS/Protobuf)
│  会话层   │        │          │
├──────────┤        ├──────────┤
│  传输层   │───────→│  传输层   │         —               段 (TCP/UDP)
├──────────┤        ├──────────┤
│  网络层   │───────→│  网络层   │       路由器             包 (IP)
├──────────┤        ├──────────┤
│ 数据链路层 │───────→│ 网络接口层 │       交换机/网卡         帧 (Ethernet)
│  物理层   │        │          │       集线器/线缆           比特
└──────────┘        └──────────┘
```

### 1.2 封装与解封装

```
发送 (封装):
应用数据 → TCP头 + 数据 → IP头 + TCP头 + 数据 → 以太网帧头 + IP头 + ... + 帧尾

接收 (解封装):
帧 → 剥离帧头帧尾 → IP包 → 剥离IP头 → TCP段 → 剥离TCP头 → 应用数据
```

## 2. IP 协议——网络层的核心

### 2.1 IP 地址分类与 CIDR

```
A 类: 0.0.0.0   ~ 127.255.255.255  ( /8 )
B 类: 128.0.0.0 ~ 191.255.255.255  ( /16 )
C 类: 192.0.0.0 ~ 223.255.255.255  ( /24 )

CIDR（无类别域间路由）:
192.168.1.0/24  → 网络部分 = 192.168.1，主机部分 = 0
/24 = 子网掩码 255.255.255.0
可用主机数 = 2^(32-24) - 2 = 254（去掉全0网络地址和全1广播地址）
```

### 2.2 路由表

```bash
$ route -n
Destination    Gateway        Genmask         Iface
0.0.0.0        192.168.1.1    0.0.0.0         eth0    ← 默认路由
192.168.1.0    0.0.0.0        255.255.255.0   eth0    ← 本地子网
```

路由查找：**最长前缀匹配**——选择子网掩码最长的匹配条目。

## 3. ARP——IP → MAC 地址的翻译

```
主机 A (192.168.1.2) 想和主机 B (192.168.1.3) 通信:
1. A 查路由表 → B 在同一子网
2. A 查 ARP 缓存 → 没有 B 的 MAC
3. A 广播 ARP 请求: "谁是 192.168.1.3？告诉我你的 MAC！"
4. B 单播 ARP 回复: "我是 192.168.1.3，MAC = aa:bb:cc:dd:ee:ff"
5. A 缓存此映射 → 封装以太网帧 → 发送
```

```bash
$ arp -a                   # 查看 ARP 缓存
$ ip neigh                 # 同上（现代 Linux）
```

## 4. ICMP——网络诊断的基础

**ICMP** = Internet Control Message Protocol。不是传数据的，是传控制信息的。

### 4.1 `ping` 的原理

```
ping 192.168.1.3:
1. 发送 ICMP Echo Request（Type=8）
2. 目标回复 ICMP Echo Reply（Type=0）
3. 计算 RTT（往返时间）
```

### 4.2 `traceroute` 的原理

```
traceroute 利用 TTL（Time To Live）:
1. 发送 TTL=1 的 UDP 包 → 第一跳路由器丢弃并返回 ICMP Time Exceeded
   → 记录第一跳 IP
2. 发送 TTL=2 → 第二跳返回 → 记录第二跳 IP
3. ...
4. 直到到达目标（目标返回 ICMP Port Unreachable）
```

```bash
$ traceroute google.com
 1  192.168.1.1  1.2ms
 2  10.0.0.1     5.4ms
 3  172.16.0.1   12.1ms
 ...
```

## 5. 网络字节序——大端 vs 小端

**网络字节序 = 大端（Big Endian）**：最高有效字节在最低地址。

```
0x12345678 在大端中的内存布局:
低地址 → 高地址
┌────┬────┬────┬────┐
│ 12 │ 34 │ 56 │ 78 │
└────┴────┴────┴────┘

x86 是小端 → 必须转换:
htonl(x)  // host to network long (32 bit)
htons(x)  // host to network short (16 bit)
ntohl(x)  // network to host long
ntohs(x)  // network to host short
```

## 小结

| 概念 | 一句话 |
|------|--------|
| TCP/IP 四层 | 应用层 → 传输层 → 网络层 → 网络接口层 |
| CIDR | IP/掩码，如 192.168.1.0/24 |
| ARP | IP → MAC，局域网内广播查询 |
| ICMP | 控制协议，ping(traceroute) 基于它 |
| TTL | 每跳减 1，到 0 丢弃——防止无限循环 |
| 大端 | 网络字节序，x86 需 `htonl`/`ntohl` 转换 |
