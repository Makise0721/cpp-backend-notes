# 01 — 进程基础：进程概念、PCB、状态模型、上下文切换

> 关联篇目：[02 线程](./02-threads.md) | [03 fork/IPC](./03-process-create-ipc.md) | [04 CPU 调度](./04-cpu-scheduling.md)

## 1. 什么是进程？

**进程 = 正在运行的程序实例**。程序是静态的（磁盘上的二进制文件），进程是动态的（加载到内存、拥有 CPU 时间片、有独立资源）。

```bash
# 同一程序可以启动多个进程
$ nginx &
$ nginx &    # 两个 nginx 进程，不同 PID
```

进程是**资源分配的基本单位**——操作系统以进程为单位分配内存、文件描述符、信号处理等资源。

## 2. 进程控制块（PCB）——进程的"身份证"

内核为每个进程维护一个 `task_struct`（Linux），核心字段：

```
task_struct {
    pid_t    pid;          // 进程 ID
    pid_t    tgid;         // 线程组 ID（= 主线程 PID）
    pid_t    ppid;         // 父进程 ID

    // 进程状态
    volatile long state;   // RUNNING / INTERRUPTIBLE / UNINTERRUPTIBLE / STOPPED / ZOMBIE

    // 内存管理
    struct mm_struct *mm;  // 地址空间描述（代码段、数据段、堆、栈）

    // 文件系统
    struct files_struct *files;  // 打开的文件描述符表

    // 调度信息
    int prio;              // 动态优先级
    int static_prio;       // 静态优先级（nice 值）

    // 信号
    struct signal_struct *signal;

    // 上下文
    struct thread_struct thread;  // 寄存器快照（上下文切换用）
};
```

## 3. 进程状态模型

### 3.1 三态模型（基本）

```
        ┌──────────┐
  创建 → │   就绪   │ ←──────────┐
        └────┬─────┘            │
             │ 被调度            │ 时间片到
             ▼                  │
        ┌──────────┐       ┌────┴─────┐
        │   运行   │ ───→  │   就绪   │
        └────┬─────┘       └──────────┘
             │ 等待事件
             ▼
        ┌──────────┐
        │   阻塞   │ ───→ 事件发生 → 就绪
        └──────────┘
```

### 3.2 五态模型（加入新建和终止）

```
新建 ─→ 就绪 ⇄ 运行 ─→ 终止
          ↑    ↓
          └── 阻塞
```

### 3.3 七态模型（加入挂起——内存不足时换出到磁盘）

```
新建 ─→ 就绪 ─→ 运行 ─→ 终止
          ↕     ↕
       就绪挂起 阻塞挂起
```

### 3.4 Linux 进程状态（`ps aux` 看到的）

| 状态码 | 含义 | 说明 |
|:---:|------|------|
| R | TASK_RUNNING | 正在运行或在运行队列中 |
| S | TASK_INTERRUPTIBLE | 可中断睡眠（等待事件，能被信号唤醒） |
| D | TASK_UNINTERRUPTIBLE | 不可中断睡眠（等 I/O，信号唤不醒——kill -9 无效！） |
| T | TASK_STOPPED | 被 SIGSTOP 暂停 |
| Z | TASK_ZOMBIE | 僵尸：已结束，父进程未回收 |

## 4. 上下文切换——进程切换的代价

### 4.1 切换时做了什么？

```
进程 A → 进程 B 的切换：
1. 保存 A 的上下文（寄存器、PC、栈指针、页表基址...）
2. 加载 B 的上下文
3. 切换页表（CR3 寄存器）
4. 刷新 TLB（部分架构需要）
5. 跳转到 B 的 PC 继续执行
```

### 4.2 开销分析

| 开销类型 | 说明 |
|----------|------|
| **直接开销** | 保存/恢复寄存器、切换内核栈 |
| **TLB 刷新** | 切换地址空间后 TLB 缓存全失效，后续访问需重新填充 |
| **CPU Cache 污染** | B 的数据会把 A 的热数据从 L1/L2 Cache 中挤出去 |
| **调度器开销** | 决定"该切到谁"本身也需要计算 |

**一次上下文切换大约消耗几微秒**（具体因架构和负载而异）。这也是为什么**线程切换比进程切换快**——线程共享地址空间，不需要切页表、不需要刷 TLB。

## 5. 进程调度相关字段

```bash
# 查看进程优先级和 nice 值
$ ps -eo pid,ni,pri,comm

# nice 值范围：-20（最高优先级）到 19（最低优先级）
# 普通用户只能提高 nice（降低优先级），root 可以任意设置
```

| 字段 | 说明 |
|------|------|
| `nice` | 用户可见的静态优先级（-20 ~ 19，越小优先级越高） |
| `pri` | 内核实际使用的动态优先级 |
| `vruntime` | CFS 调度器的虚拟运行时间（见 [04 CPU 调度](./04-cpu-scheduling.md)） |

## 6. 进程的组织结构

```
Linux 用双向链表组织所有进程：

init_task (PID 1, systemd/init)
   ├→ task_struct (PID 2) → task_struct (PID 3) → ...

所有进程组成一棵树（父子关系）:
systemd (1)
├─ sshd (234)
│  └─ sshd (567) ← 某次 SSH 连接的子进程
│     └─ bash (568)
├─ nginx (300)
│  ├─ nginx worker (301)
│  └─ nginx worker (302)
└─ mysqld (400)
```

## 小结

| 概念 | 一句话 |
|------|--------|
| 进程 | 程序运行实例，资源分配基本单位 |
| PCB | 内核中描述进程的数据结构（`task_struct`） |
| 五态 | 新建/就绪/运行/阻塞/终止 |
| Linux 状态 | R(运行) / S(中断睡眠) / D(不可中断) / T(暂停) / Z(僵尸) |
| 上下文切换 | 保存/恢复上下文，切换页表 → TLB 全刷新，开销几微秒 |
| 进程间关系 | 树形父子结构，孤儿进程被 init 收养 |
