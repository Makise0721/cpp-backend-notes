# 08 — 中断与异常：IDT、上半部/下半部、softirq/tasklet/workqueue

> 关联篇目：[01 进程基础（上下文切换）](./01-process-basics.md) | [10 系统调用](./10-system-basics.md)

---

## 1. 中断 vs 异常

| | 中断（Interrupt） | 异常（Exception） |
|------|------|------|
| 来源 | **外部**（硬件设备） | **内部**（CPU 执行指令时） |
| 触发 | 异步（随时可能发生） | 同步（与指令执行绑定） |
| 例子 | 网卡收包、键盘按键、时钟 | 除零、缺页、断点 |
| 处理上下文 | 中断上下文 | 进程上下文 |

### 1.1 异常的三个子类

| 类型 | 例子 | 行为 |
|------|------|------|
| **Fault**（故障） | 缺页异常 | 修复后可重试（返回出错的指令） |
| **Trap**（陷阱） | 断点、系统调用 | 返回下一条指令 |
| **Abort**（中止） | 硬件错误、双重故障 | 无法恢复，进程终止 |

## 2. 中断描述符表（IDT）

**IDT 是一张表，存储了每个中断/异常向量的处理入口地址**。

```
中断向量号 → IDT[向量号] → 门描述符（中断门 / 陷阱门 / 任务门）
                           → 处理函数入口地址 + 特权级
```

```
常见向量号：
0   — 除零错误
3   — 断点 (int3)
13  — 一般保护性错误 (GPF)
14  — 缺页异常 (#PF)
80  — Linux 系统调用 (32 位 int 0x80)
```

---

## 3. 中断上下文——不能睡觉

中断处理程序运行在**中断上下文**中。此时：

- ❌ **不能睡眠**（没有进程上下文，调度器不知道该唤醒谁）
- ❌ **不能调用可能阻塞的函数**（`mutex_lock`、`kmalloc(GFP_KERNEL)` 等）
- ❌ **不能访问用户态内存**（当前不与任何进程关联）
- ✅ 只能用自旋锁（`spin_lock`）

---

## 4. 上半部 vs 下半部

中断处理必须**快**——关中断期间其他中断得不到响应。所以把处理分成两半：

```
中断发生
  │
  ▼
上半部（Top Half）：直接响应硬件，做最少必要的工作
  │  例：读取网卡寄存器，将数据包放入接收队列
  │  → 关中断运行，必须极快（微秒级）
  │
  ▼
下半部（Bottom Half）：做剩下的耗时操作
  │  例：解析数据包、IP 处理、TCP 协议栈、唤醒等待进程
  │  → 开中断运行，可以被抢占
  │
  ▼
完成
```

## 5. 下半部三种机制对比

| | **softirq** | **tasklet** | **workqueue** |
|------|:---:|:---:|:---:|
| 运行上下文 | 中断上下文 | 中断上下文 | **进程上下文**（可睡眠！） |
| 并发 | 同类型可在多核并发 | 同类型串行（自动加锁） | 并发（每个 work 独立） |
| 抢占 | 不能被抢占 | 不能被抢占 | 可被抢占 |
| 使用场景 | 高频、性能敏感 | 大多数驱动 | 需要睡眠的操作 |
| 例子 | 网络收发（NET_RX/TX）、定时器 | 网卡驱动下半部 | 磁盘 I/O 完成处理 |

```c
// tasklet 示例
void my_tasklet_func(unsigned long data) { /* 下半部 */ }
DECLARE_TASKLET(my_tasklet, my_tasklet_func, 0);
tasklet_schedule(&my_tasklet);  // 在上半部中调度

// workqueue 示例（可以睡眠！）
struct work_struct my_work;
INIT_WORK(&my_work, my_work_func);
schedule_work(&my_work);
```

### 5.1 选择决策

```
下半部需要睡眠吗？
├─ 是 → workqueue
└─ 否
    ├─ 高频、性能敏感 → softirq（网络栈）
    └─ 一般驱动 → tasklet（更简单）
```

## 6. 中断屏蔽与优先级

```
x86 中断优先级（部分）:
TRAP（除零等） > NMI > INTR（可屏蔽中断）

通过 cli/sti 或 IF 标志位控制可屏蔽中断：
cli  → 关中断（禁止响应可屏蔽中断）
sti  → 开中断
```

现代 Linux 尽量避免长时间关中断——**中断线程化**（threaded interrupts）将一部分中断处理移到内核线程中，进一步减少关中断时间。

---

## 小结

| 概念 | 一句话 |
|------|--------|
| 中断 | 外部硬件异步触发 |
| 异常 | CPU 执行指令同步触发（fault/trap/abort） |
| IDT | 中断描述符表，向量号→处理函数 |
| 中断上下文 | 不能睡眠、不能用 mutex、不能用可能阻塞的函数 |
| 上半部 | 极快处理，关中断 |
| 下半部 | 耗时处理：softirq（高频）、tasklet（驱动）、workqueue（可睡眠） |
