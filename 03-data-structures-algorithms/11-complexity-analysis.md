## 1. 时间复杂度——算法运行时间随输入规模的增长

### 1.1 常见复杂度排序（从快到慢）

```
O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

| 复杂度 | n=10 | n=100 | n=10⁶ | 典型算法 |
|--------|------|-------|-------|----------|
| O(1) | 1 | 1 | 1 | 数组索引 |
| O(log n) | ~3 | ~7 | ~20 | 二分查找 |
| O(n) | 10 | 100 | 10⁶ | 顺序查找 |
| O(n log n) | ~30 | ~700 | ~2×10⁷ | 归并/快排 |
| O(n²) | 100 | 10⁴ | 10¹² | 冒泡/选择/插入 |
| O(2ⁿ) | 1024 | ~10³⁰ | — | 子集枚举 |

### 1.2 怎么算？

1. **找到最内层循环的执行次数**
2. **忽略常数系数和低阶项**
3. **取增长最快的项**

```cpp
// O(n²)
for (int i = 0; i < n; ++i)     // n 次
    for (int j = 0; j < n; ++j) // n 次
        work();                   // 总共 n² 次

// O(n + m)——多个变量分开考虑
for (int i = 0; i < n; ++i) work();
for (int j = 0; j < m; ++j) work();

// O(log n)——每次规模减半
while (n > 1) { n /= 2; work(); }
```

## 2. 空间复杂度——算法需要的额外内存

```cpp
// O(1) 空间 —— 原地算法
void reverse(vector<int>& arr) {
    int l = 0, r = arr.size() - 1;
    while (l < r) swap(arr[l++], arr[r--]);
}

// O(n) 空间 —— 辅助数组
vector<int> prefixSum(vector<int>& arr) {
    vector<int> prefix(arr.size() + 1, 0);
    for (int i = 0; i < arr.size(); ++i)
        prefix[i+1] = prefix[i] + arr[i];
    return prefix;  // O(n) 额外空间
}

// 递归栈空间也要算！
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // O(n) 递归栈空间
}

// 尾递归 → O(1) 栈空间（编译器优化后）
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);  // 尾调用
}
```

## 3. 最好 / 最坏 / 平均复杂度

```cpp
// 快速排序：
// 最好:  O(n log n) — pivot 总是中位数
// 平均:  O(n log n) — 随机 pivot
// 最坏:  O(n²)      — pivot 总是最值（已排序数组 + 固定选最右）
```

| 分析级别 | 含义 | 何时用 |
|----------|------|--------|
| 最好 | 最理想输入下的复杂度 | 几乎不用（太乐观） |
| 最坏 | 最差输入下的复杂度 | 保证上限，面试**必答** |
| 平均 | 所有可能输入的期望值 | 真实性能预期 |

## 4. 摊还分析（Amortized Analysis）

**一次操作可能很慢，但一系列操作的平均很快。**

### 4.1 聚合分析——vector 的 push_back

```cpp
vector<int> v;
for (int i = 0; i < n; ++i) v.push_back(i);

// 扩容发生在 size = 1, 2, 4, 8, ..., 2^k
// 每次扩容复制现有元素：1 + 2 + 4 + ... + n/2 ≈ n
// n 次 push_back 总开销 ≈ n(push) + n(复制) = 2n
// 每次 push_back 均摊 = O(1)
```

### 4.2 记账法

```
每次 push_back 存 3 块钱：
- 1 块钱付自己的插入
- 2 块钱存起来，未来扩容时替自己和非扩容插入的元素付搬迁费
```

### 4.3 势能法

定义势函数 Φ，操作的实际代价 + ΔΦ = 摊还代价。略，面试中很少深入。

### 4.4 常见摊还复杂度

| 操作 | 摊还 | 单次最坏 |
|------|:---:|:---:|
| vector push_back | O(1) | O(n) |
| 并查集 find | O(α(n)) | O(log n) |
| 哈希表插入 | O(1) | O(n)（rehash） |

## 5. 主定理（Master Theorem）

用于分析**分治递归算法**的复杂度：

```
T(n) = a × T(n/b) + f(n)

a = 子问题个数
b = 子问题规模缩小因子
f(n) = 分解 + 合并的开销
```

### 5.1 三种情况

设 **d = log_b(a)**（即 a 写成 b^d）：

```
Case 1: f(n) = O(n^(d-ε))     → T(n) = Θ(n^d)
        （合并开销 小于 递归开销）

Case 2: f(n) = Θ(n^d × log^k n) → T(n) = Θ(n^d × log^(k+1) n)
        （合并开销 等于 递归开销）

Case 3: f(n) = Ω(n^(d+ε)) + 正则条件 → T(n) = Θ(f(n))
        （合并开销 大于 递归开销）
```

### 5.2 实战案例

```
归并排序: T(n) = 2T(n/2) + O(n)
  a=2, b=2, d=log₂2=1
  f(n) = O(n) = Θ(n¹)  → Case 2 (k=0)
  T(n) = Θ(n log n)

二分查找: T(n) = T(n/2) + O(1)
  a=1, b=2, d=log₂1=0
  f(n) = O(1) = Θ(n⁰)  → Case 2 (k=0)
  T(n) = Θ(log n)

Strassen 矩阵乘: T(n) = 7T(n/2) + O(n²)
  a=7, b=2, d=log₂7≈2.81
  f(n) = O(n²) = O(n^(d-0.81))  → Case 1
  T(n) = Θ(n^2.81)
```

### 5.3 主定理速查

| 算法 | 递推式 | 复杂度 |
|------|--------|:---:|
| 二分查找 | T(n) = T(n/2) + O(1) | O(log n) |
| 二叉树遍历 | T(n) = 2T(n/2) + O(1) | O(n) |
| 归并排序 | T(n) = 2T(n/2) + O(n) | O(n log n) |
| 快速排序（平均） | T(n) = 2T(n/2) + O(n) | O(n log n) |
| 建堆 | T(n) = 2T(n/2) + O(log n) | O(n) |
| 最近点对 | T(n) = 2T(n/2) + O(n) | O(n log n) |

## 6. 复杂度分析实战技巧

```cpp
// 多层循环 → 乘积
for (i: 0..n)          // O(n)
    for (j: 0..n)      // O(n)
        → O(n²)

// 不独立的循环 → 分析执行次数
for (i: 0..n)
    for (j: i..n)
        → n + (n-1) + ... + 1 = O(n²)

// 规模每次减半 → 对数
while (n > 1) { n /= 2; }  → O(log n)

// 递归画出递归树 → 每层开销 × 层数
// 或用主定理直接套公式
```

## 小结

| 概念 | 一句话 |
|------|--------|
| 大 O | 上界，最坏情况，忽略常数和低阶 |
